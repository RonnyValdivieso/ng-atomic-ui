Button Component

Componente fundamental para acciones del usuario. Incluye múltiples variantes, tamaños, estados de carga y soporte completo para iconos con PrimeIcons.

Características Principales

4 Variantes

primary, secondary, danger, success

3 Tamaños

small, medium, large

Estados

disabled, loading, focused

Características

Iconos, labels, eventos click

Variantes Disponibles

Primary variant="primary"
Secondary variant="secondary"
Danger variant="danger"
Success variant="success"

Tamaños Disponibles

Small size="small"
Medium size="medium"
Large size="large"

Botones con Iconos

Añadir icon="pi pi-plus"
Eliminar icon="pi pi-trash"
Descargar icon="pi pi-download"
Confirmar icon="pi pi-check"

Botones Solo Icono

Solo icono
Icono pequeño
Icono grande

Estados del Botón

Normal Estado normal
Disabled [disabled]="true"
Loading [loading]="true"

Casos de Uso Comunes

Acciones Principales

Guardar Enviar Confirmar

Acciones Secundarias

Cancelar Volver Ver Detalles

Acciones Destructivas

Eliminar Bloquear

Mejores Prácticas

Recomendado

  • • Usar variant="primary" para acciones principales
  • • Combinar iconos con labels descriptivos
  • • Usar loading state para operaciones asíncronas
  • • Mantener consistencia en tamaños por contexto
  • • Usar variant="danger" para acciones destructivas

Evitar

  • • Múltiples botones primary en la misma vista
  • • Botones sin feedback visual al hacer clic
  • • Labels muy largos que rompan el diseño
  • • Iconos sin significado claro
  • • Botones disabled sin explicación al usuario

Ejemplos de Código

<!-- Botón básico -->
<app-button variant="primary" (clicked)="handleClick()">
  Guardar
</app-button>

<!-- Botón con icono -->
<app-button 
  variant="success" 
  icon="pi-check" 
  (clicked)="confirm()">
  Confirmar
</app-button>

<!-- Botón con estado de carga -->
<app-button 
  variant="primary" 
  [loading]="isLoading" 
  [disabled]="isLoading"
  (clicked)="submit()">
  Enviar
</app-button>
